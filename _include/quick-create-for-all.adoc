[role="lead"]
Você pode implantar Cloud Volumes ONTAP na Amazon Web Services (AWS) e Microsoft Azure usando `Quick create` para uma implantação mais rápida, tanto em configurações de nó único quanto de alta disponibilidade (HA). Esse processo simplificado reduz as etapas de implantação em comparação com o método avançado, definindo automaticamente os valores padrão em uma única página e minimizando a navegação.

.Detalhes do sistema
. *Conta/credenciais do provedor de nuvem*: Os detalhes da conta e da assinatura são preenchidos automaticamente com base no seu agente do Console. Se você tiver várias contas ou assinaturas, selecione aquela que deseja usar. Se um agente do Console estiver indisponível, você será solicitado a  https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html["criar um agente de console"^].
. *Nome*: Insira um nome para o seu sistema Cloud Volumes ONTAP (também chamado de nome do cluster). O Console usa o nome do sistema (cluster) para nomear o sistema Cloud Volumes ONTAP e a instância da VM. Ele também usa o nome como prefixo para o grupo de segurança predefinido, caso você selecione essa opção.
. *Credenciais do ONTAP*: Estas são as credenciais da conta de administrador do cluster Cloud Volumes ONTAP. Você pode usar essas credenciais para se conectar ao Cloud Volumes ONTAP por meio do ONTAP System Manager ou da ONTAP CLI. Você pode manter o nome de usuário padrão _admin_ ou alterá-lo para um nome de usuário personalizado.
. *Tags*: As tags são os metadados dos seus recursos da AWS ou do Azure. O Console adiciona as tags à instância do Cloud Volumes ONTAP e a cada recurso associado à instância. Você pode adicionar até 15 tags na interface de usuário ao criar um sistema Cloud Volumes ONTAP e, em seguida, adicionar mais após a criação. Para obter mais informações, consulte:

.Implantação e configuração
. *Tipo de implantação/topologia*: Selecione o tipo de implantação que deseja usar, nó único, alta disponibilidade (HA) em uma única zona de disponibilidade (AZ) ou HA em várias AZs.
. *Configuração de rede*: Insira as informações de rede que você registrou na planilha de planejamento para regiões, VNets e sub-redes.
+
.. *Região*: Por padrão, a região da conta de nuvem associada aos recursos é selecionada.
.. *Zona de disponibilidade*: Você pode selecionar a zona de disponibilidade com base no tipo de implantação escolhido e se a região selecionada oferece suporte à zona de disponibilidade.
.. *Sub-rede*: selecione uma sub-rede para a região.
+

.Alta disponibilidade da AWS
Se você selecionou uma configuração de alta disponibilidade (HA), insira as seguintes informações:

.HA em AZ único
... *Acesso do mediador*: especifique as informações de acesso do mediador.  O mediador é uma instância separada que monitora a integridade do par HA e fornece quorum em caso de falha.  Forneça o nome do par de chaves para permitir que a instância do mediador se conecte ao serviço AWS EC2 e selecione o método de conexão.


.HA em várias AZ
... *Zonas de disponibilidade e mediador*: selecione as zonas de disponibilidade (AZs) para cada nó, o mediador e as sub-redes correspondentes onde você deseja implantar o par Cloud Volumes ONTAP HA.
... *IPs flutuantes*: se você escolher várias AZs, especifique os endereços IP flutuantes para os serviços NFS e CIFS e para o gerenciamento de cluster e SVM.  Os endereços IP devem estar fora do bloco CIDR para todas as VPCs na região.  Para obter detalhes adicionais, consultelink:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/reference-networking-aws.html#requirements-for-ha-pairs-in-multiple-azs["Requisitos de rede da AWS para Cloud Volumes ONTAP HA em várias AZs"^] .
... *Acesso do mediador*: especifique as informações de acesso do mediador.  O mediador é uma instância separada que monitora a integridade do par HA e fornece quorum em caso de falha.  Forneça o nome do par de chaves para permitir que a instância do mediador se conecte ao serviço AWS EC2 e selecione o método de conexão.
... *Tabelas de rotas*: se você escolher várias zonas de disponibilidade, selecione as tabelas de rotas que incluem rotas para os endereços IP flutuantes.  Se você tiver mais de uma tabela de rotas, é importante selecionar as tabelas de rotas corretas.  Caso contrário, alguns clientes podem não ter acesso ao par Cloud Volumes ONTAP HA.  Para obter mais informações sobre tabelas de rotas, consulte o http://docs.aws.amazon.com/AmazonVPC/latest/UserGuide/VPC_Route_Tables.html["Documentação da AWS: Tabelas de rotas"^] .

ifdef::azure[]

No Azure, a seção *Visão geral* exibe os recursos que serão alocados quando o processo de implantação for concluído.

endif::azure[]

ifdef::aws[]

Na _AWS_, imediatamente após você criar o sistema Cloud Volumes ONTAP, o NetApp Console inicia uma instância de teste na VPC especificada para verificar a conectividade. Se bem-sucedido, o Console termina imediatamente a instância e então inicia a implantação do sistema. Se o Console não conseguir verificar a conectividade, a criação do sistema falhará. A instância de teste é uma `t2.nano` (para locação de VPC padrão) ou uma `m3.medium` (para locação de VPC dedicada).

endif::aws[]

.Depois que você terminar
O Console implanta o sistema Cloud Volumes ONTAP .  Você pode acompanhar o progresso na página Auditoria.

Se você encontrar algum problema ao implantar o sistema Cloud Volumes ONTAP, revise a mensagem de falha. Você também pode selecionar *Recriar ambiente* no sistema para continuar com a implantação usando o assistente de modo  `Advanced`.


CAUTION: Após a conclusão do processo de implantação, não modifique as configurações do Cloud Volumes ONTAP geradas pelo sistema no portal da AWS ou Azure, especialmente as tags do sistema. Quaisquer alterações feitas nessas configurações podem levar a comportamentos inesperados ou perda de dados.
